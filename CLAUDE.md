## Project Overview

Gravel Road Finder - A Python tool that queries OpenStreetMap data via Overpass API to discover and score gravel cycling roads. The tool analyzes unpaved roads based on surface quality, maintenance, traffic, and other factors to help cyclists find premium gravel routes.

## Commands

### Setup
```bash
# Install dependencies
poetry install

# Install with satellite imagery support (Phase 2 - optional)
poetry install --extras satellite
```

### Running the Tool
```bash
# Basic usage (default: Wrocław, 25km radius, 0.1km min length)
poetry run python find_gravel_roads.py

# Named search (creates output/wroclaw/)
poetry run python find_gravel_roads.py --name wroclaw

# Custom location and filters
poetry run python find_gravel_roads.py --center 52.2297,21.0122 --radius 30 --min-score 70 --min-length 1.0

# Large area search (increase timeout)
poetry run python find_gravel_roads.py --radius 100 --timeout 300

# Force fresh query from OSM (bypass cache)
poetry run python find_gravel_roads.py --name wroclaw --force-query

# Increase retry attempts for unreliable connections
poetry run python find_gravel_roads.py --name wroclaw --force-query --max-retries 5

# Specific output formats
poetry run python find_gravel_roads.py --formats geojson,html
```

## Architecture

### Entry Point
- `find_gravel_roads.py` - CLI script using argparse that orchestrates the entire workflow:
  1. Query OSM via Overpass API
  2. Score each road segment
  3. Filter by length, score, and tier
  4. Export to multiple formats

### Core Package: `gravel_roads/`

**`osm_query.py`** - OpenStreetMap data retrieval
- `get_gravel_roads(center, radius_km, timeout)` - Main query function
- Overpass QL query targets:
  - `highway=track` with gravel surfaces or `tracktype=grade1/grade2`
  - `highway=path` with gravel surfaces (bicycle-friendly)
  - `highway=service` with gravel surfaces (forest/cycling paths)
  - Emergency access roads with good smoothness
  - Gravel residential roads
- Returns list of road dictionaries with OSM tags and calculated length
- Uses Haversine formula for distance calculations

**`scoring.py`** - Premium scoring algorithm (0-100 points)
- `calculate_premium_score(road)` - Returns total_score, tier, and breakdown
- **6 scoring categories**:
  - Surface quality (25 pts): fine_gravel > gravel > compacted > ground > unpaved > dirt
  - Maintenance/tracktype (25 pts): grade1 > grade2 > grade3 > unknown > grade4 > grade5
  - Smoothness (20 pts): excellent > good > intermediate > bad > rough > very_bad...
  - Traffic level (15 pts): path > bicycle-only > forestry/agricultural > general access
  - Width (10 pts): ≥3m > 2-3m > unknown > <2m
  - Access (5 pts): bicycle=designated/yes > access=yes > permissive > private
- **Tier thresholds**: Premium ≥80, Good ≥65, Acceptable ≥50, Poor <50

**`storage.py`** - Data export
- `save_geojson(roads, filepath)` - GeoJSON with LineString geometries and full properties
- `save_csv(roads, filepath)` - Tabular format with coordinates and score breakdown

**`visualization.py`** - Interactive maps
- `create_interactive_map(roads, center, output_path)` - Folium HTML map
- Color scheme (heat map): Red=Premium, Orange=Good, Yellow=Acceptable, Blue=Poor
- Popups show detailed road info, score breakdown, and OSM links
- Multiple base layers: OpenStreetMap, OpenTopoMap

## Key Implementation Details

### Coordinate Systems
- OSM/Python internal: `(latitude, longitude)` tuples
- GeoJSON export: `[longitude, latitude]` arrays (standard GeoJSON format)

### Default Values
- Center: Wrocław, Poland (51.1079, 17.0385)
- Radius: 25km (use `--radius` to change)
- Min length: 0.1km (filters very short segments)
- Timeout: 180s for Overpass API
- Target grade: `tracktype=grade2` (well-maintained unpaved roads)

### Overpass API
- Uses `overpy` library for Overpass QL queries
- Query searches `around:radius` from center point
- Returns OSM ways with full tag data
- May timeout on very large areas (increase `--timeout` if needed)

### Caching & Retry Logic
- **Caching**: By default, loads from existing GeoJSON (fast, no API calls)
  - Use `--force-query` to fetch fresh data from OSM
  - Only HTML is regenerated by default (allows quick styling updates)
- **Retry logic**: Automatic retry with exponential backoff (5s, 15s, 45s)
  - Handles "Server load too high", rate limits, and timeouts
  - Default: 3 attempts (configurable with `--max-retries`)
  - Non-retryable errors fail immediately

## Modifying Behavior

### Adjust Scoring Weights
Edit `gravel_roads/scoring.py` - modify point values in scoring dictionaries:
- `surface_scores` (25 points total)
- `tracktype_scores` (25 points total)
- `smoothness_scores` (20 points total)
- Traffic, width, access scoring logic (15+10+5 = 30 points)

### Change OSM Query
Edit `gravel_roads/osm_query.py` - modify Overpass QL query in `get_gravel_roads()`:
- Add/remove highway types
- Adjust surface filters
- Include additional OSM tags

### Add Export Formats
Extend `gravel_roads/storage.py` - create new `save_*()` functions

### Customize Map Appearance
Edit `gravel_roads/visualization.py`:
- `tier_colors` dictionary for color scheme
- Popup HTML template
- Base map layers and zoom level
